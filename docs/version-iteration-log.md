# 分诊智能体版本迭代记录

## V1.0 原型版

日期：2026-03-13

### 当前已完成能力

- FastAPI 后端接口可运行
- `/docs` 和 `/demo` 可用于交互测试
- 已接入阿里云百炼 API

### V1.0 当前已知问题

#### 1. 否定表达容易被误提取成阳性特征

问题示例：

- 患者说：“不会出现什么呼吸不顺之类的”
- 系统可能仍提取出：`呼吸不顺 / 呼吸困难`

问题本质：

- 当前特征提取仍有较多“关键词命中”逻辑
- 对“没有 / 不会 / 并不 / 否认”这类否定范围的处理还不够完整
- 导致模型或规则在遇到“包含症状词，但语义是否定”的句子时，可能误判为症状存在

影响：

- 会造成风险标记偏多
- 会让分诊等级偏保守
- 可能浪费急诊或高优先级资源

下一版修复方向：

- 为高风险症状增加“否定表达识别”层
- 对 `呼吸困难 / 胸痛 / 出血 / 外伤` 等关键症状做否定优先判定
- 在提取轨迹中区分：
  - `positive_feature`
  - `negated_feature`
  - `uncertain_feature`

#### 2. 外伤部分仍存在 bug

问题示例：

- 患者按要求回复：“没有创伤” / “没有外伤”
- 后续结果里仍可能出现：
  - `创伤规则：存在外伤或车祸相关信息`

问题本质：

- 当前规则引擎里仍存在对原始文本的关键词匹配
- 文本里虽然是否定表达，但仍包含“外伤 / 创伤 / 车祸”等关键词
- 如果规则层继续按字符串包含关系判断，仍可能误触发外伤规则

影响：

- 会把普通病例误判为创伤相关病例
- 会影响推荐科室与风险标记

下一版修复方向：

- 外伤判断改为“结构化字段优先”，即优先使用 `trauma_history`
- 当患者明确回复“没有外伤 / 没有创伤”时，规则引擎不再对原句重复做正向创伤触发
- 为外伤增加显式状态：
  - `True`
  - `False`
  - `Unknown`

#### 3. 部分普通症状仍可能被分流得过于保守

问题示例：

- 患者主诉喉咙痛、吞咽不适
- 如果伴随特征抽取不够细，可能被系统分流得过于紧张

问题本质：

- 当前规则对“普通上呼吸道 / 耳鼻喉症状”和“真正危险气道问题”的区分还不够细
- 特征粒度还不够丰富，例如：
  - 普通咽痛
  - 吞咽疼痛
  - 吞咽困难
  - 声音嘶哑
  - 呼吸困难
  - 喘鸣

下一版修复方向：

- 增加更细的 ENT 症状层级
- 将“普通咽痛”与“危险气道信号”分开处理
- 减少不必要的急诊推荐

### V1.0 当前设计定位

本版本定位是：

- 先跑通“患者输入 -> 特征提取 -> 规则兜底 -> 分诊输出”的最小闭环
- 重点是流程通、接口通、会话通、规则框架通
- 暂不追求所有自然语言表达都能精准理解

也就是说，V1.0 是“可运行原型”，不是“高精度临床可用版”。

## V1.1 计划修复版

### 目标

- 降低误提取
- 修复否定表达 bug
- 修复外伤误判 bug
- 提高普通症状与危险症状的区分能力

### 重点任务

- 增加否定词规则
- 给高风险特征增加极性标注
- 优化外伤判定逻辑
- 补更多中文口语表达样本
- 增加“普通 ENT 症状 vs 危险气道症状”规则

## V2.0 推荐增强版

### 目标

- 从“推荐就诊入口”升级到“推荐具体诊室 / 科室”

### 计划能力

- 推荐耳鼻喉科门诊、普通内科门诊、儿科门诊、急诊等具体诊室方向
- 支持更细的门诊入口映射，例如：
  - 咽痛 -> 耳鼻喉科门诊
  - 儿童发热 -> 儿科门诊
  - 腹痛伴呕吐 -> 儿科 / 普外分流
  - 胸痛伴呼吸困难 -> 急诊

### 需要补充的数据与规则

- 科室映射表
- 年龄分层规则
- 主诉到诊室的映射策略
- 医院实际门诊结构适配

## V3.0 患者身份与访问版

### 目标

- 在会话系统中加入患者身份、记录管理与访问能力

### 计划能力

- 为患者分配个人 ID
- 支持用患者 ID 查找历史会话
- 支持会话持久化存储
- 支持同一患者多次来访记录
- 支持交接单归档

### 需要新增模块

- 患者主表
- 会话表
- 分诊结果表
- 登录 / 访问控制
- SQLite 或更正式的数据库持久化

## 后续版本建议路线

建议后续迭代顺序：

1. 先完成 V1.1
   - 先把误提取、否定表达、外伤误判修掉

2. 再做 V2.0
   - 把“推荐入口”升级成“推荐诊室”

3. 最后做 V3.0
   - 加患者 ID、记录、访问和持久化

## 一句话总结

V1.0 已经完成了“能跑通”的分诊原型，但当前最需要优先修复的是：

- 否定表达误提取
- 外伤误判
- 普通症状被分流过于保守

而中期最值得做的增强是：

- 推荐具体诊室
- 患者 ID 与历史记录管理
